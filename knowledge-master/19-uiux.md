# UI/UX

User Interface & User Experience — how products look, feel, and function from the user's perspective.

## UX Principles

**Usability Heuristics (Nielsen):**
1. Visibility of system status
2. Match between system and real world
3. User control and freedom
4. Consistency and standards
5. Error prevention
6. Recognition rather than recall
7. Flexibility and efficiency of use
8. Aesthetic and minimalist design
9. Help users recognize, diagnose, and recover from errors
10. Help and documentation

**User Research:** Observe what people do, not what they say. Interviews, usability testing, analytics, surveys.

**Information Architecture:** Structure content logically. Card sorting, tree testing, sitemaps.

**User Flows:** Map every path through the product. Identify friction points and drop-offs.

## UI Principles

- **Hierarchy:** Visual weight communicates importance. Size, color, position, spacing.
- **Consistency:** Same patterns for same purposes. Reduces learning curve.
- **Affordance:** Visual cues that suggest how to interact. Buttons look clickable.
- **Feedback:** Every action gets a visible reaction. States: hover, active, disabled, loading, error, empty.
- **Accessibility:** WCAG 2.2 AA minimum. Color contrast, keyboard navigation, screen reader support, focus indicators.

## Visual Design Elements

**Typography:** Max 2-3 typefaces. Readable body text (16px+). Proper line-height (1.5 body, 1.2 headings).

**Color:** Primary, secondary, accent, neutral, semantic (success, warning, error, info). 60-30-10 rule.

**Spacing:** Consistent scale (4px or 8px grid). White space is not wasted space.

**Layout:** Responsive grids. Breakpoints at content, not devices.

## Motion Design

Functional animation: transitions, feedback, spatial orientation. Duration 200-300ms standard. Easing: ease-out for exits, ease-in-out for emphasis.
