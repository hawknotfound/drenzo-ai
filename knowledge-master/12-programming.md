# Programming

Core programming knowledge — paradigms, patterns, principles, and practical skills.

## Fundamental Concepts

**Data Structures:** Arrays, linked lists, hash tables, trees (binary, BST, tries), graphs, heaps, stacks, queues.

**Algorithms:** Sorting (quick, merge, heap), search (binary, BFS, DFS), dynamic programming, greedy, divide & conquer.

**Time Complexity:** Big O notation. O(1) constant, O(log n) logarithmic, O(n) linear, O(n log n), O(n²) quadratic.

**Paradigms:** Imperative, declarative, functional, OOP, procedural. Each has strengths.

## Best Practices

- **DRY (Don't Repeat Yourself):** Extract repeated logic
- **KISS:** Keep it simple — simpler is easier to maintain
- **SOLID (OOP):** Single responsibility, Open-closed, Liskov substitution, Interface segregation, Dependency inversion
- **Separation of concerns:** Different layers for different responsibilities
- **Error handling:** Fail gracefully, log meaningfully, never swallow errors silently

## Language Families

**C-family (C, C++, C#, Java, JS/TS, Go):** Curly brace syntax, similar control flow
**Scripting (Python, Ruby, PHP, Perl):** Dynamic typing, interpreted, rapid development
**Functional (Haskell, Elixir, Clojure, F#):** Immutability, pure functions, declarative style

## Tools & Workflow

- Version control (Git) — non-negotiable
- Package managers — npm, pip, cargo, gem
- Testing — unit, integration, e2e. TDD when appropriate
- Debugging — systematic elimination of variables
- Documentation — for why, not what
